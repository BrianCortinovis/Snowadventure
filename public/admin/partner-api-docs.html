1. Autenticazione

Tutte le richieste API richiedono l'header X-API-Key con la chiave API fornita al momento dell'attivazione della partnership.

La chiave API e personale e riservata. Non condividerla pubblicamente e non esporla nel codice frontend. Usa sempre chiamate server-side.
// Header obbligatorio in ogni richiesta
X-API-Key: sa_partner_abc123def456...

Esempio con cURL

curl -H "X-API-Key: sa_partner_abc123def456" \
     https://tuodominio.com/api/partner/experiences

Esempio con JavaScript (server-side)

const response = await fetch('https://tuodominio.com/api/partner/experiences', {
  headers: {
    'X-API-Key': process.env.SNOW_ADVENTURE_API_KEY
  }
});
const experiences = await response.json();

2. Base URL

https://tuodominio.com/api/partner

Tutti gli endpoint sono relativi a questo percorso base. Sostituisci tuodominio.com con il dominio effettivo del sito Snow Adventure.

3. Lista Esperienze

Ottieni l'elenco delle esperienze disponibili per il tuo account partner. Vedrai solo le esperienze a cui sei stato autorizzato.

GET /api/partner/experiences
Restituisce tutte le esperienze attive a cui il partner ha accesso.

Risposta

[
  {
    "id": "uuid",
    "name": "Orobic Sunset Tour",
    "slug": "sunset",
    "description": "Escursione in motoslitta al tramonto...",
    "duration": "2 ore",
    "price_cents": 14900,
    "max_sleds": 6,
    "photo_url": "images/sunset.jpg",
    "is_active": true
  },
  ...
]

Campi risposta

CampoTipoDescrizione
idstringIdentificativo univoco esperienza
namestringNome dell'esperienza
slugstringSlug URL (usare per disponibilita e prenotazioni)
descriptionstringDescrizione completa
durationstringDurata (es. "2 ore")
price_centsintegerPrezzo in centesimi. Potrebbe differire se hai prezzi personalizzati
max_sledsintegerNumero massimo di motoslitte per sessione
photo_urlstringURL immagine dell'esperienza

4. Verifica Disponibilita

Controlla la disponibilita degli slot per una specifica esperienza e periodo.

GET /api/partner/availability
Restituisce gli slot disponibili per l'esperienza e il periodo richiesto.

Parametri query

ParametroTipoObbligatorioDescrizione
experiencestringSiSlug dell'esperienza (es. sunset)
fromstringSiData inizio (formato YYYY-MM-DD)
tostringSiData fine (formato YYYY-MM-DD)

Esempio richiesta

GET /api/partner/availability?experience=sunset&from=2026-03-15&to=2026-03-22

Risposta

[
  {
    "date": "2026-03-15",
    "time": "17:00",
    "available_sleds": 4,
    "max_sleds": 6,
    "price_cents": 14900
  },
  {
    "date": "2026-03-15",
    "time": "18:30",
    "available_sleds": 6,
    "max_sleds": 6,
    "price_cents": 14900
  },
  ...
]
Vengono restituiti solo gli slot con almeno 1 motoslitta disponibile. Gli slot bloccati o completi non appaiono.

5. Crea Prenotazione

Crea una nuova prenotazione per conto di un cliente. La prenotazione viene creata in stato pending e deve essere confermata dal team Snow Adventure.

POST /api/partner/bookings
Crea una nuova prenotazione per il cliente.

Body (JSON)

CampoTipoObbligatorioDescrizione
experience_slugstringSiSlug dell'esperienza
datestringSiData prenotazione (YYYY-MM-DD)
timestringSiOrario slot (HH:MM)
nomestringSiNome del cliente
cognomestringSiCognome del cliente
emailstringSiEmail del cliente
phonestringNoTelefono del cliente
num_sledsintegerSiNumero di motoslitte richieste
num_peopleintegerSiNumero totale di persone
notesstringNoNote aggiuntive

Esempio richiesta

POST /api/partner/bookings
Content-Type: application/json
X-API-Key: sa_partner_abc123def456

{
  "experience_slug": "sunset",
  "date": "2026-03-20",
  "time": "17:00",
  "nome": "Marco",
  "cognome": "Rossi",
  "email": "marco.rossi@email.com",
  "phone": "+39 333 1234567",
  "num_sleds": 2,
  "num_people": 4,
  "notes": "Cliente dell'hotel, camera 204"
}

Risposta (201 Created)

{
  "booking_ref": "SA-20260320-A1B2",
  "status": "pending",
  "experience": "Orobic Sunset Tour",
  "date": "2026-03-20",
  "time": "17:00",
  "amount_cents": 29800,
  "commission_cents": 2980,
  "message": "Prenotazione creata con successo"
}

Campi risposta

CampoTipoDescrizione
booking_refstringRiferimento univoco della prenotazione
statusstringStato iniziale (pending)
amount_centsintegerTotale in centesimi
commission_centsintegerLa tua commissione in centesimi
La commissione viene calcolata automaticamente in base alla percentuale concordata nel tuo contratto partner.

7. Commissioni e Prezzi

Ogni partner ha una percentuale di commissione configurata dall'amministratore. Le commissioni vengono calcolate automaticamente su ogni prenotazione.

Come funziona

  • La commissione viene applicata sul totale della prenotazione
  • Il campo commission_cents nella risposta indica la commissione maturata
  • I partner possono avere prezzi personalizzati per specifiche esperienze, diversi dal listino pubblico
  • Le commissioni vengono regolate mensilmente secondo gli accordi contrattuali

Esempio calcolo

Prenotazione: 2 motoslitte x 149€ = 298€
Commissione partner (10%): 29,80€
Netto Snow Adventure: 268,20€

8. Gestione Errori

L'API restituisce errori in formato JSON con un campo error descrittivo.

HTTP StatusSignificatoEsempio
400Richiesta non validaParametri mancanti o formato errato
401Non autenticatoAPI Key mancante o non valida
403Non autorizzatoPartner disattivato o esperienza non consentita
404Non trovatoEsperienza o slot non esistente
409ConflittoSlot non piu disponibile (overbooking)
429Troppe richiesteRate limit superato
500Errore serverErrore interno

Esempio risposta errore

{
  "error": "Slot non disponibile per la data e l'orario richiesto",
  "code": "SLOT_UNAVAILABLE"
}

9. Rate Limiting

Per garantire la stabilita del servizio, le richieste API sono limitate a:

  • 100 richieste al minuto per API key
  • 5.000 richieste al giorno per API key

In caso di superamento, riceverai un errore 429 Too Many Requests. Implementa un meccanismo di retry con backoff esponenziale.
